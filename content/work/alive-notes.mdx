## Problem

Most note apps are dumb buckets. You write things down. They sit there. The thing you wrote down to remember slowly becomes the thing you forgot you wrote down. Intention dies inside an inbox that never forwards.

## Build

Alive Notes is a mobile-first OS for human intention. Capture a thought, the system reads it, classifies it, and routes it. A grocery item goes to your list. A weekly intention surfaces every Sunday. A long-term goal turns into a check-in cadence. The notes become alive in the literal sense: they keep showing up at the right moment.

Built with Expo SDK 54 and React Native for one shared codebase across iOS and Android. Zustand v5 for local state, Supabase for sync, OpenAI for the classification layer. Hand-tuned haptics, font stack, and animations because mobile-first is a craft, not a checkbox.

## Outcome

A mobile app that does what your notes app should have done all along. Used daily for capture, weekly for review, quarterly for orientation.

## Stack

Expo SDK 54, React Native, Zustand v5, Supabase, OpenAI. Native iOS feel, written once, shipped to both stores.

## What this is, generally

A working example of what happens when a mobile app stops being a database with a UI and starts being a system with a posture. Same architecture works for journaling, fitness tracking, sales pipeline, prayer apps, learning systems. The premise transfers.
