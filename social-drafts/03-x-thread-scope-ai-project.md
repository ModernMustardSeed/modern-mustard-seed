# X Thread: Scope an AI Project in 90 Minutes

**Hook (post 1/7):**
Most AI projects fail at scope, not execution.

Wrong workflow picked. Built for the wrong customer. Open-ended engagement drags into next quarter.

The fix is a tighter front door.

This is the exact 90-minute scoping conversation I run 👇

**2/7:**
Block 1 (0-15 min): THE JOB.

Write down, in one sentence, the job the AI is hired to do.

❌ "Use AI to improve customer experience"
✅ "Auto-triage 100% of inbound support emails into the right category and surface a draft reply"

If you can't write it in one sentence, you don't have a job. You have a vibe.

**3/7:**
Block 2 (15-35 min): MAP THE CURRENT WORKFLOW.

Step by step. Who, what tool, what input, what output, how long.

End with a flowchart on paper.

The AI candidates are the steps where a human is doing repetitive cognitive work.

This step takes longer than you expect. Don't skip it.

**4/7:**
Block 3 (35-55 min): THE LEVERAGE MATRIX.

Plot each candidate:

Vertical: monthly impact (real $ or hours)
Horizontal: AI fit (1-5)

Top-right = your roadmap.
Top-left = "AI can't yet, revisit in 6 mo."
Bottom-right = AI can but impact small. Skip.

Focus on 3 max.

**5/7:**
Block 4 (55-75 min): BUILD SEQUENCE.

The order rule:

1. Highest impact + lowest risk first
2. Most downstream value second
3. Moonshot last (only if #1 and #2 ship)

Most teams reverse this. They build the exciting thing first and run out of momentum before the valuable thing ever ships.

**6/7:**
Block 5 (75-90 min): ONE-PAGE SPEC.

For the first candidate, write a one-pager:

- One-sentence job
- Input + output
- Reasoning layer (model + prompt + guardrails)
- Human override (when AI hands back)
- Success metric (one number)
- Cut list
- Timeline (weeks, not months)

If it doesn't fit on one page, scope is too big.
